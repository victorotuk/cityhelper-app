# Landing Page Best Practices — Desktop vs Mobile (2024–2025)

Research summary + visual reference. See `assets/landing-best-practices-desktop-vs-mobile.png` for the comparison.

**Implemented in Nava:** Landing hero uses research-backed layout (desktop: two-column, trust card; mobile: single column, one full-width CTA, "Learn more" hidden). Copy: "Never miss a deadline again" + "Track renewals, visas, taxes, trusts, parking tickets — almost anything with a deadline. Get reminders. Soon: AI that helps you file applications."

---

## What the research says

### Desktop
- **Above the fold:** Headline + subheadline + one primary CTA + product visual (screenshot/video)
- **Trust badge** at top (e.g. "Trusted by 10K+ users")
- **Headline formula:** [Who it's for] + [Specific outcome] + [Time/qualifier]
- **Avoid:** Abstract words ("unlock," "transform"), multiple CTAs, long feature lists
- **5-second test:** Can someone explain what it does and who it's for after 5 seconds?

### Mobile
- **Single column** — no multi-column
- **Short headline** — one line, benefit-focused
- **One CTA** — full-width, in thumb zone (bottom 60% of screen)
- **Minimal above fold** — headline, one line, CTA. No long feature lists.
- **Sticky CTA** — optional, for long pages
- **Tap targets** — min 44px

### Conversion stats
- Forms: 4 fields → ~120% more conversions than 11 fields
- Mobile converts ~8% lower than desktop (often due to slow load, poor UX)
- Page load under 2.5s; each extra second ≈ 7% lower conversion
- Social proof (testimonials with names, photos) can lift conversions 19–34%

### What to avoid
- Too much promo copy
- Multiple CTAs competing for attention
- Feature lists before the value prop
- Long paragraphs above the fold
